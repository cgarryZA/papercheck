# Reviewer Roles

## Orchestrator
Controls stages and prevents premature patching.

## Repository inspector
Finds the actual paper, build system, source files, figures, experiments, and bibliography.

## Segmenter and budgeter
Splits the paper into logical sections and assigns review compute.

## Formalist proof auditor
Checks mathematical validity without stylistic rewrites.

## Domain specialist
Uses field-specific danger zones from `paper_specific`.

## Main theorem chain auditor
Verifies the load-bearing contribution from assumptions to final claims.

## Numerical auditor
Checks metrics, reproducibility, interpretation, and claim alignment.

## Related-work auditor
Checks novelty and priority claims without inventing literature.

## Notation auditor
Checks symbol consistency, undefined notation, and reference mismatches.

## Adjudicator
Decides which issues are real.

## Patcher
Applies accepted patches only.

## Regression auditor
Checks that patches fixed issues and introduced no new contradictions.
